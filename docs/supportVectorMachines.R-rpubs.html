Introduction

  • previous chapters have introduced multiple methods for classification
    • linear discriminant analysis, logistic regression (Cp. 4)
    • classification trees, boosting and bagging (Cp. 8)
  • Support vector machine (SVM) is a group of classification methods, which are based on the concept of margin
    • maximal margin classifier (MMC) simplest one and requires linear separation between different classes
    • support vector classifier (SVC) “softens” the requirements of MMC
    • SVM most general and applicable for non-linear problems

On the notation

  • \( j=1,\ldots,p \), where \( p \) is the number of dimensions (\( p = 2 \) in most examples)
  • \( i = 1,\ldots,n \), where \( n \) is the number of observations
  • \( X = [X_1, X_2,\ldots,X_p]^T \) a point in p dimensions
  • \( x^* = [x^*_1, x^*_2,\ldots,x^*_p]^T \) a test observation
  • \( y_i \) is the class of a point

9.1 Definition of hyperplane

  • for a \( p \)-dimensional space, defined as a flat affine
    subspace with \( p-1 \) dimensions
  • a point \( (X_1,\ldots,X_p)^T \) is on a hyperplane if
    \( \beta_0 + \beta_1X_1 + \ldots + \beta_pX_p = 0 \) holds
  • for example, \( 1 + 1X_1 + 2X_2 + 3X_3 = 0 \)
    is a 2-d hyperplane for a set of points
    \( X=(X_1,X_2,X_3)^T \) in 3-dimensional space

9.1.1 Why the concept of hyperplane is useful?

  • when \( \beta_0 + \beta_1X_1 + \ldots + \beta_pX_p > 0 \)
    point \( X=(X_1,\ldots,X_p)^T \) lies on one side of
    the hyperplane
  • Conversely, if \( \beta_0 + \beta_1X_1 + \ldots + \beta_pX_p < 0 \)
    point \( X=(X_1,\ldots,X_p)^T \) is on the other side
    of the hyperplane
  • if this separating hyperplane exists,
    it can be used to construct a classifier

9.1.2 Separating hyperplane

  • separates two or more classes from each other
  • consider training observations \[ x_i= \begin{pmatrix} x_{i1} \\ \vdots \\ x_{ip} \end{pmatrix} \], where \( i = 1,\ldots,n \)
  • for observation \( x_i \), the class \( y_i \) gets value \( \color{blue}{1} \) if
    \( (\beta_0 + \beta_1x_{i1} + \beta_2x_{i2} + ... + \beta_px_{ip}) > 0 \)
    and \( y_i=\color{red}{-1} \)
    \( (\beta_0 + \beta_1x_{i1} + \beta_2x_{i2} + ... + \beta_px_{ip}) < 0 \)
  • the distance of \( x_i \) from the separating hyperplane
    gives also valuable information
  • essentially gives a linear decision boundary

9.1.3 The maximal marginal classifier (MMC)

  • problem: an infinite number of
    separating hyperplanes can be chosen
  • solution: find the maximal margin hyperplane (MMH)
    • a hyperplane with the largest minimum
      distance (margin) to the training observations
  • MMH depends only on the support vectors
    (grey lines)

9.1.4 Construction of MMC

  • maximise \( M \) with parameters \( \beta_1,\beta_2,\ldots,M \) subject to \[ \sum^{p}_{j=1} \beta^2_j = 1 \] and \[ y_i(\beta_0 + \beta_1x_{i1} + \beta_2x_{i2} + \ldots + \beta_px_{ip}) \ge M ~ \forall i = 1,\ldots,n \]
  • \( M \) is the size of the margin
  • test observation is then classified according to \( f(x_i^{*}) = \beta_{0} + \beta_{1}x^{*}_{1} + \ldots + \beta_{p}x^{*}_{p} \)

9.1.5 Shortcomings of MMC

  • MMC works only when the separating hyperplane exists
    • when classes overlap, there is no solution
      to the optimisation problem when \( M>0 \)
  • MMC can be sensitive to individual observations

9.2 Support vector classifier (soft margin classifier)

  • does not require that all observations are on the correct side
    of the margin or even the separating hyperplane
  • the optimisation problem is the following
  • Maximise \( M \) with parameters \( \beta_1,\beta_2,\ldots,\epsilon_1,\ldots,\epsilon_n,M \) subject to \[ \sum^{p}_{j=1} \beta^2_j = 1, \] \[ y_i(\beta_0 + \beta_1x_{i1} + \beta_2x_{i2} + \ldots + \beta_px_{ip}) \ge M(1-\epsilon_i) ~ \forall i = 1,\ldots,n, \] \[ \epsilon_i\ge0, \quad \sum^n_{i=1} \epsilon_i \le C \]

9.2 Slack variables

  • \( \epsilon_1, \ldots, \epsilon_n \) so-called slack variables
    • position of the ith observation relative to the hyperplane and margins
    • \( \epsilon_i = 0 \), observation is on the right side of the margin
    • \( \epsilon_i \gt 0 \), observation is on the wrong side of the margin
    • \( \epsilon_i \gt 1 \), observation is on the wrong side of the hyperplane

9.2 Tuning parameter C

  • \( C \), tuning parameter, controls the bias-variance trade-off
    • \( C=0 \) gives the MMH -> risk of over-fitting
    • \( C>0 \), no more than \( C \) observations allowed on the wrong side of the hyperplane
    • use cross-validation to find the optimal value

9.3.1 Non-linear decision boundaries

  • SVC fails if class boundaries are non-linear
  • we could add quadratic \( (X_i^2) \), qubic (\( X_i^3 \)) or interaction \( (X_iX_j) \) terms etc.
    • the resulting decision boundary (or hyperplane) linear in the enlarged feature space but not in the original!
  • too many features make computations infeasible
    • support vector machine (SVM) avoids this issue using kernels

9.3.1 Support vector machine

  • SVM can be represented as \[ f(x) = \beta_0 + \underset{i \in S}{\sum}\alpha_i K(x,x_i) \]
    • \( \alpha_i \) parameters for each support point in \( S \) selected from training observations
    • \( K \) is the kernel function (generalisation of inner product)
  • compare to \( f(x) = \beta_0 + \beta_1x_1 + \ldots + \beta_px_p \)

9.3.1 Kernel examples

  • SVM uses kernels \( K(x_i,x_{i'}) \) for classification
    • \( K(x_{i},x_{i'}) = \sum^p_{j=1}(x_{ij}x_{i'j}) \), linear kernel with normal inner product
    • \( K(x_{i},x_{i'}) = (1+\sum^p_{j=1}(x_{ij}x_{i'j}))^d \), polynomial kernel
    • \( K(x_{i},x_{i'}) = \exp(-\gamma\sum^p_{j=1}(x_{ij}-x_{i'j}))^2 \), radial kernel, sometimes \( \gamma=1/(2\sigma) \)

9.3.1 Why to use kernels?

  • a mapping \( \varphi \) from p-dimensional space to other (higher dimensional) feature space \[ K(x_{i},x_{i'}) = \varphi(x_{i})\varphi(x_{i'}) \]
  • if \( K(x_{i},x_{i'}) = (1 + x_{i}x_{i'})^2 \) with \( x_i=(x_{i1},x_{i2}) \) and \( x_i'=(x_{i'1},x_{i'2}) \), then \[ K(x_{i},x_{i'}) = (1+ x_{i1}x_{i'1} + x_{i2}x_{i'2})^2 \\ = 1 + x_{i1}^2x_{i'1}^2 + x_{i2}^2x_{i'2}^2 + 2x_{i1}x_{i'1} + 2x_{i2}x_{i'2} + 2x_{i1}x_{i'1}x_{i2}x_{i'2} \]
    • this isthe inner product of \( (1,x_{i1}^2,x_{i2}^2,\sqrt(2)x_{i1},\sqrt(2)x_{i2},\sqrt(2)x_{i1}x_{i2}) \) and
      \( (1,x_{i'1}^2,x_{i'2}^2,\sqrt(2)x_{i'1},\sqrt(2)x_{i'2},\sqrt(2)x_{i'1}x_{i'2}) \)
    • mapping to 6-dimensional feature space, no need to calculate explicitly

9.4 More than two classes

  • SVM do not extend easily to K>2 classes

  • one-versus-one:

    • construct \( K(K-1)/2 \) SVMs for each pair of classes
      -> classify test observations with each SVM
      -> assign the test observation to the class to which it was assigned most frequently by the pairwise SVMs

  • one-versus-all:

    • construct \( K \) SVMs which compare a class (labeled as +1) against the other \( K-1 \) classes (labeled as -1)
      -> test observation \( x^{*} \) is assigned to the class for which \( \beta_{0k} + \beta_{1k} x^{*}_{1} + \beta_{2k}x^{*}_2+\ldots+\beta_{pk}x^{*}_{p} \) is largest

9.5 Relationship to the logistic regression

  • rewrite the SVC optimisation problem for \( f(X) = \beta_0 + \beta_1X_1+\ldots+\beta_pX_p \) as \[ \underset{\beta_0,\beta_1,\ldots,\beta_p}{\mathrm{minimize}} \left\{ \sum^{n}_{i=1} \left[0, 1 - y_{i} f(x_i) \right] + \lambda \sum^{p}_{j=1} \beta^{2}_{j} \right\} \]
  • the first term in the brackets is called the hinge loss and the second one is the penalty term
  • when \( \lambda \) is small (large), \( \beta_j \) are large (small) and more (less) violations to the margin are tolerated
  • only support vectors affect the fitting procedure in SVC
